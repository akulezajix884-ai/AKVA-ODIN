# Review 1 — Первичный аудит

## Общая оценка: 4/10

**Обоснование:** Файл index.html отсутствует после 5 попыток ожидания (2.5 минуты). Без HTML-файла невозможно оценить семантику, accessibility, SEO, 3D-placeholder, интерактивные компоненты (бургер-меню, FAQ-аккордеон, слайдер), Intersection Observer, подключение шрифтов. Имеющиеся style.css и copy.md выполнены качественно, но сайт без HTML не является сайтом. Оценка включает бонус за хорошее качество CSS и текстов.

## КОД

### Критические проблемы

1. **[index.html] ФАЙЛ ОТСУТСТВУЕТ** — Главный файл проекта не был сгенерирован. Это блокирующая проблема. Без HTML невозможно:
   - Проверить валидность HTML5
   - Проверить наличие `<div id="3d-hero-placeholder">`
   - Проверить семантические теги (`<header>`, `<main>`, `<section>`, `<footer>`, `<nav>`)
   - Проверить подключение Google Fonts
   - Проверить meta description, OG-теги
   - Проверить aria-labels, alt-атрибуты
   - Проверить Intersection Observer (JS)
   - Проверить работу бургер-меню, FAQ-аккордеона, слайдера отзывов
   - Проверить H1-H6 иерархию
   - Проверить skip-to-content ссылку
   - Проверить JSON-LD structured data
   --> **Рекомендация: Сгенерировать index.html немедленно. Это приоритет номер один.**

2. **[style.css:66] Отсутствует --font-mono** — В дизайн-системе определена переменная `--font-mono: 'JetBrains Mono', monospace` для цен и технических данных. В CSS custom properties блоке style.css эта переменная не объявлена. Карточка товара в design-system.md использует `font-family: var(--font-mono)` для цен.
   --> **Рекомендация: Добавить `--font-mono: 'JetBrains Mono', monospace;` в :root блок style.css.**

3. **[style.css] Отсутствует подключение Google Fonts** — CSS не содержит `@import` для Google Fonts. Без index.html неясно, подключаются ли шрифты через `<link>`. Если HTML не содержит подключения, Montserrat и Inter не загрузятся.
   --> **Рекомендация: Убедиться, что в index.html есть `<link>` на `https://fonts.googleapis.com/css2?family=Montserrat:wght@500;600;700;800&family=Inter:wght@400;500;600&display=swap` или добавить @import в CSS. Также добавить preconnect.**

### Некритические замечания

4. **[style.css:1221-1226] Цены используют --font-heading вместо --font-mono** — В `.product-card__price` задан `font-family: var(--font-heading)`, хотя дизайн-система предписывает `font-family: var(--font-mono)` для цен (design-system.md, строка 289).
   --> **Рекомендация: Заменить на `font-family: var(--font-mono)` для соответствия дизайн-системе.**

5. **[style.css:1432-1433] FAQ question :focus без visible outline** — `.faq-item__question:focus` устанавливает `outline: none`, что нарушает accessibility. Пользователи с клавиатурой не увидят фокус.
   --> **Рекомендация: Заменить на `outline: 2px solid var(--color-primary-400); outline-offset: 2px;` или использовать `:focus-visible` вместо `:focus`.**

6. **[style.css:86-88] Отсутствуют переменные сетки** — В design-system.md определены `--grid-columns: 12`, `--grid-gutter: 24px`, `--grid-gutter-mobile: 16px`. В style.css объявлен только `--container-max: 1200px`.
   --> **Рекомендация: Добавить пропущенные переменные для полноты системы.**

7. **[style.css:505-509] Breakpoint переменные отсутствуют** — Дизайн-система определяет `--bp-mobile`, `--bp-tablet`, `--bp-desktop`, `--bp-wide`. Они не объявлены в style.css. Хотя CSS custom properties нельзя использовать в @media напрямую, они полезны как справочные значения и для JS.
   --> **Рекомендация: Добавить breakpoint переменные как reference-комментарии или для использования в JS.**

8. **[style.css:1451-1453] FAQ max-height захардкожена** — `.faq-item.open .faq-item__answer` использует `max-height: 300px`. Если ответ длиннее 300px (а FAQ #1 и #3 в copy.md довольно длинные), текст будет обрезан.
   --> **Рекомендация: Увеличить до `max-height: 600px` или использовать JS для динамического расчёта scrollHeight.**

9. **[style.css:529-567] Wave divider использует fill на path, но оригинал в design-system.md использует currentColor** — design-system.md определяет волну через `fill="currentColor"` и управление цветом через `.wave-divider { color: ... }`. В style.css используются `.wave-divider--primary svg path { fill: ... }`, что требует другой HTML-разметки (path без fill="currentColor").
   --> **Рекомендация: Согласовать подход — либо currentColor в SVG + color в CSS, либо прямой fill на path.**

10. **[style.css:1860-1864] Wide breakpoint меняет container max-width** — На 1440px+ контейнер расширяется до 1320px, хотя дизайн-система указывает `--container-max: 1200px`. Это расхождение.
    --> **Рекомендация: Удалить переопределение или обновить дизайн-систему, чтобы включить wide-вариант.**

11. **[style.css] Нет стилей для формы заказа** — Дизайн-система содержит компоненты `.form-input`, `.form-label`, `.form-select` (design-system.md, строки 358-394). В style.css эти стили полностью отсутствуют. Форма заказа — ключевой элемент шага 5 воронки (Action).
    --> **Рекомендация: Добавить стили форм из дизайн-системы.**

12. **[style.css] Отсутствует стиль `.btn-secondary` на тёмном фоне для CTA-блока** — В hero-секции есть переопределение `.hero__buttons .btn-secondary` для белого текста на тёмном фоне, но в CTA-блоке (#cta) аналогичное переопределение отсутствует, хотя по site-structure.md CTA-блок тоже на тёмном фоне.
    --> **Рекомендация: При необходимости добавить переопределение для btn-secondary в cta-block.**

## ДИЗАЙН

### Критические проблемы

1. **[design-system.md] Не хватает состояний для :focus-visible** — Дизайн-система описывает hover, active состояния кнопок, но не определяет focus-visible стили. Это пробел в accessibility-проектировании.
   --> **Рекомендация: Добавить описание focus-visible стилей для всех интерактивных компонентов.**

### Некритические замечания

2. **[design-system.md] JetBrains Mono не указан в Google Fonts URL** — На строке 101 дана ссылка на Google Fonts только для Montserrat и Inter. JetBrains Mono (`--font-mono`) отсутствует в URL подключения. Шрифт доступен на Google Fonts, но не включён.
   --> **Рекомендация: Добавить `&family=JetBrains+Mono:wght@500;700` в URL Google Fonts.**

3. **[design-system.md] Нет определения success/error/warning цветов** — Для формы заказа нужны цвета валидации: зелёный (успех), красный (ошибка). Их нет в палитре.
   --> **Рекомендация: Добавить семантические цвета: `--color-success`, `--color-error`, `--color-warning`.**

4. **[design-system.md] Компонент сравнительной таблицы не описан** — Funnel.md шаг 2 предполагает сравнительную таблицу "водопровод vs АкваВита", но в дизайн-системе нет описания компонента таблицы.
   --> **Рекомендация: Добавить описание компонента таблицы сравнения.**

### Несоответствия

1. **КРИТИЧЕСКОЕ: index.html отсутствует** — Невозможно проверить соответствие HTML с design-system, site-structure, copy.md и funnel.md. Все проверки cross-file consistency заблокированы.

   - site-structure.md: "Чистая вода с доставкой до двери в Одинцово"
   - cta-texts.md (вариант A): "Чистая вода для вашей семьи — с доставкой до двери"
   - copy.md: "Вода из горных источников — к вашей двери"
   --> Нет единого источника истины для главного заголовка сайта.

3. **Подзаголовок hero — цена "от 199 руб" в site-structure.md** — Минимальная цена по каталогу (copy.md) — 460 руб за "Тверскую ДеЛюкс". Цена "от 199 руб" нигде больше не фигурирует. Либо это цена за что-то другое (разовая бутыль меньшего объёма?), либо ошибка.

7. **CSS не содержит --font-mono** — Дизайн-система объявляет три шрифта, CSS реализует только два.

8. **Заголовок секции FAQ** — site-structure.md: "Часто задаваемые вопросы". copy.md: нет отдельного заголовка секции (начинается сразу с вопросов). Заголовок подразумевается.

9. **Заголовок CTA-блока — расхождение:**
   - site-structure.md: "Закажите доставку воды прямо сейчас"
   - copy.md: "Попробуйте воду, к которой не захочется добавлять фильтр"
   - cta-texts.md (вариант A): "Закажите воду — привезём сегодня"

10. **style.css не содержит стили формы** — design-system.md определяет `.form-input`, `.form-label`, `.form-select`. В style.css их нет. Форма заказа — ключевой конверсионный элемент шага 5.
