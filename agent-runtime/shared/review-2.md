**Обоснование:** index.html создан и содержит все ключевые секции, семантику, JSON-LD, OG-теги, Intersection Observer, бургер-меню, FAQ-аккордеон, слайдер отзывов, sticky CTA, WhatsApp-виджет. CSS полноценный, mobile-first. Тексты качественные. Остаются проблемы с accessibility (FAQ focus), отсутствием --font-mono, расхождениями текстов между HTML и copy.md, и рядом мелких недочётов. ## КОД

### Критические проблемы

1. **[style.css:66] --font-mono отсутствует в :root** — Дизайн-система определяет `--font-mono: 'JetBrains Mono', monospace` для цен и технических данных. Переменная не объявлена в CSS. Цены в `.product-card__price` используют `var(--font-heading)` вместо `var(--font-mono)`.
   --> Добавить `--font-mono: 'JetBrains Mono', monospace;` в :root. Изменить `.product-card__price` (строка 1222) на `font-family: var(--font-mono)`.

2. **[index.html:20] JetBrains Mono не подключён в Google Fonts** — URL подключения содержит только Montserrat и Inter. Шрифт для цен не загрузится.
   --> Изменить строку 20 на: `https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600&family=JetBrains+Mono:wght@500;700&family=Montserrat:wght@500;600;700;800&display=swap`

3. **[style.css:1431-1433] FAQ :focus удаляет outline без замены** — `.faq-item__question:focus { outline: none; }` нарушает accessibility (WCAG 2.4.7). Пользователи клавиатуры не видят фокус.
   --> Заменить на: `.faq-item__question:focus-visible { outline: 2px solid var(--color-primary-400); outline-offset: 2px; }`

4. **[style.css] Отсутствуют стили формы заказа** — design-system.md определяет `.form-input`, `.form-label`, `.form-select`. В style.css их нет. CTA-блок (index.html:736-768) содержит только ссылку-кнопку, а не форму. Форма заказа — ключевой конверсионный элемент шага 5 воронки (Action). Без формы вся воронка "обрывается" — пользователь может только позвонить.
   --> Добавить форму заказа (имя, телефон, адрес) в секцию #cta в index.html. Добавить стили .form-input, .form-label в style.css.
