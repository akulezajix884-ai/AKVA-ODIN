Текущий сайт клиента: https://www.akvavita-mo.ru/

Мы делаем современный редизайн сайта компании по продаже воды. Сайт выглядит как из 2010-х — нужен свежий, премиальный, конверсионный дизайн. На главную страницу в будущем будут добавлены 3D-модели (заглушки под них нужно предусмотреть уже сейчас). Воронка должна включать элементы психологии продаж — пошаговое вовлечение пользователя.

## Задачи для команды:

### Agent 1 (design-architect)

Роль: дизайн-директор + UI/UX архитектор.
Задача:

- Изучи текущий сайт https://www.akvavita-mo.ru/ — зафиксируй разделы, навигацию и элементы которые стоит сохранить.
- Составь дизайн-систему для водной тематики: цветовая палитра (синий, прозрачный, белый, акценты), типографика, отступы, компоненты.
- Придумай структуру страниц на основе текущего сайта: Hero → О воде → Преимущества → Воронка → Продукты → Отзывы → CTA → Футер.
- В Hero-секции предусмотри контейнер `<div id="3d-hero-placeholder">` под будущую Three.js / Spline 3D-модель.
- Сохрани дизайн-систему в `agent-runtime/shared/design-system.md`.
- Сохрани структуру страниц в `agent-runtime/shared/site-structure.md`.

### Agent 2 (frontend-developer)

Роль: senior frontend-разработчик.
Задача:

- Дождись появления файлов `agent-runtime/shared/design-system.md` и `agent-runtime/shared/site-structure.md`.
- Собери полноценный HTML+CSS+JS сайт на основе дизайн-системы.
- Используй CSS custom properties (переменные), Flexbox/Grid, анимации на scroll (Intersection Observer).
- В Hero вставь `<div id="3d-hero-placeholder" class="hero__3d">` с комментарием `<!-- Three.js / Spline mount point -->`.
- Сделай мобильную адаптацию (mobile-first).
- Сохрани результат в `agent-runtime/outputs/index.html` и `agent-runtime/outputs/style.css`.

### Agent 3 (conversion-psychologist)

Роль: психолог продаж + воронка.
Задача:

- Спроектируй пошаговую психологическую воронку для продажи воды: осознание проблемы → интерес → доверие → желание → действие.
- Каждый шаг воронки — отдельная секция на странице с триггером (боль, факт, социальное доказательство, срочность, CTA).
- Опиши какие психологические принципы применяются на каждом шаге (Cialdini, AIDA, Jobs To Be Done).
- Сохрани структуру воронки в `agent-runtime/shared/funnel.md`.
- Напиши заготовки CTA-кнопок и заголовков для каждого шага воронки в `agent-runtime/shared/cta-texts.md`.

### Agent 4 (copywriter)

Роль: копирайтер премиального уровня, специализация — FMCG и здоровый образ жизни.
Задача:

- Дождись появления `agent-runtime/shared/funnel.md` и `agent-runtime/shared/site-structure.md`.
- Напиши все тексты для сайта: Hero-заголовок, подзаголовки, блоки преимуществ, факты о воде, описания продуктов, отзывы (3 шт.), FAQ (5 вопросов), футер.
- Тексты должны быть живыми, с образами воды, природы, чистоты и здоровья. Без клише типа "лучшая вода в мире".
- Используй структуру из `funnel.md` — тексты должны усиливать каждый шаг воронки.
- Сохрани все тексты в `agent-runtime/outputs/copy.md`.

### Agent 5 (qa-reviewer)

Роль: критик и ревьюер — проверяет код, дизайн и тексты в несколько итераций.
Задача:

- Работай в цикле. После каждой итерации проверяй все файлы в `agent-runtime/outputs/` и `agent-runtime/shared/`.
- **Итерация 1** — первичный аудит: проверь `index.html`, `style.css`, `copy.md`, `design-system.md`, `funnel.md`. Зафиксируй все проблемы в `agent-runtime/shared/review-1.md` по категориям: Код / Дизайн / Тексты / Воронка.
- **Итерация 2** — после того как агенты внесли правки по review-1, повтори проверку. Сохрани в `agent-runtime/shared/review-2.md`.
- **Итерация 3** — финальная проверка. Убедись что все замечания устранены. Напиши итоговый вердикт в `agent-runtime/shared/final-verdict.md`: что готово к релизу, что требует ручной доработки.
- В каждом review указывай конкретно: файл, строка или секция, проблема, рекомендация.
- Критерии проверки:
  - Код: валидный HTML, работающий CSS, мобильная адаптация, наличие 3D-заглушки
  - Дизайн: соответствие водной теме, современность, визуальная иерархия
  - Тексты: нет клише, есть образы, соответствие воронке
  - Воронка: все 5 шагов на месте, психологические триггеры работают

## ВАЖНО:

- Агенты 1, 2, 3, 4 стартуют параллельно и ведут основную разработку.
- Агент 5 (qa-reviewer) стартует сразу, но первую проверку начинает только когда появятся файлы в `agent-runtime/outputs/`.
- После каждого review агент 5 уведомляет остальных через `agent-runtime/messages/` о найденных замечаниях.
- Агенты 1–4 читают сообщения от qa-reviewer и вносят правки. Цикл повторяется минимум 2 раза.
- Все результаты — в `agent-runtime/outputs/`.
- Финал: index.html + style.css + copy.md + final-verdict.md = готовый проверенный сайт.
